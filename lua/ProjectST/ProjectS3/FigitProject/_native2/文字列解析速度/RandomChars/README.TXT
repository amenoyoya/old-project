【ソフト紹介】―――――――――――――――――――――――――――――――

　このツールは、ランダムな文字列で構成される任意サイズのファイルを複数個生成
できるツールです。プログラムのテストなどで、任意サイズのファイルが必要な場合
や、ランダムな文字列が大量に必要な場合などに活用できます。

　ランダムな文字列は、次の3種類から選択できます。

　1. テキストデータ（英数字のみで構成）
　2. バイナリデータ（制御文字も含むバイナリデータで構成）
　3. ゼロ（NULL）

　1ファイルのサイズ（文字数）や、1回の操作で生成するファイル数は自由に指定可
能です。また、複数ファイルを同時に生成する場合は、ファイル名もランダムなもの
になります。

　フォーマットしたリムーバブルメディアから、ファイルを復元することを不可能に
するために、ランダムな文字列でメディア内を埋め尽くすことにも活用できるでしょう。

　コマンドラインから各種設定を指定して起動することもできるので、バッチファイ
ルやその他のスクリプトから呼び出して実行することも可能です。その際は、処理完
了後に自動的に終了するため、スクリプトの一部としてシームレスに活用できます。

　本ソフトウェアはレジストリを使用しません。
　アンインストールするには、プログラムを格納しているフォルダを削除するだけです。

【簡単な使用方法】―――――――――――――――――――――――――――――

　単純なインターフェイスですので、説明がなくてもお分かりいただけると思いま
すが、念のため下記に簡単に説明しておきます。

　起動するには、RandomChars.exe を実行します。
　表示されるダイアログから、以下の点を指定します。

　　・生成先フォルダ
　　・生成ファイル名（複数作成する場合は「ランダム」にチェックを入れます）
　　・生成サイズ（最小1Bytes〜最大サイズの制限はありません）
　　・生成個数（ファイル名を自分で指定した場合は自動的に「1」になります）
　　・生成内容（テキスト／バイナリ／ゼロから選べます）

　上記の項目をすべて指定したら、「出力」ボタンをクリックします。
　これで、指定個数・指定サイズのファイルが生成されます。

　生成処理中は、進行状況を示すダイアログが表示されます。
　生成中はCPUパワーを消費しますので、他の処理との同時実行はあまりお勧めできま
せん。「一時停止」ボタンを押せば、一時的に処理を停止できます。停止中は、CPUパ
ワーは消費しません。「中止」ボタンですぐに全処理を中止することもできます。
　（「中止」ボタンの代わりに、[ESC]キーを押すことでも中止できます。）

　数百MBを越えるファイルの生成には、かなり時間がかかる可能性がありますので、
ご注意下さい。また、アクセス速度の遅いメディアに書き込む場合、「中止」ボタンや
「一時停止」ボタンの反応が遅くなる場合があります。その際は、ボタンをクリックし
てから、しばらく他の操作をせず待って下さい。そのうちクリックが反映されるはずで
す。

　このソフトウェアの設定は、RandomChars.ini というファイル名で、プログラムファ
イルと同じフォルダに保存されます。特に削除しても問題ありません。

　※ステータスバーに説明が表示されます※
　ウインドウ内の各入力欄にマウスポインタを乗せれば、ステータスバーに説明が表
示されます。その説明も参考にして下さい。


【コマンドラインからの使用方法】――――――――――――――――――――――

　このソフトウェアは、コマンドラインから『も』実行できます。バッチファイルや
その他のスクリプトから直接呼び出して使用する際などにご活用下さい。
　すべての設定は、パラメータで指定できます。以下のような書式で起動できます。

　RandomChars.exe [/A] [/F:(フォルダ名)] [/N:(ファイル名)] [/R] [/S:(サイズ)]
                     [/U:(単位)] [/T:(個数)] [/C:(生成内容種別)] [/I] [/H]

　パラメータの意味は、以下の通りです。

　/F:(フォルダ名)　:ファイルを生成するフォルダ名を指定します。
　　　　　　　　　　存在するフォルダしか指定できません。
　　　　　　　　　　※空白文字を含むフォルダは指定できないので注意して下さい。
　/N:(ファイル名)　:生成するファイル名を指定します。（「/R」と排他的に指定）
　　　　　　　　　　生成個数が2個以上の場合は指定しても意味がありません。
　/R　　　　　　　 :ファイル名をランダムに決定します。（「/N」と排他的に指定）
　/S:(サイズ)　　　:生成するファイル1個あたりのサイズを数値で指定します。
　/U:(単位)　　　　:「/S」で指定したサイズの単位を以下の番号で指定します。
　　　　　　　　　　　0 = Bytes
　　　　　　　　　　　1 = KB
　　　　　　　　　　　2 = MB
　　　　　　　　　　　3 = GB
　　　　　　　　　　上記以外の番号は無効です。無効な番号を指定した場合や、指
　　　　　　　　　　定を省略した場合は「KB」を指定したと解釈されます。
　/T:(個数)　　　　:生成するファイルの個数を数値で指定します。
　/C:(生成内容種別):生成するランダムデータの内容を以下の番号で指定します。
　　　　　　　　　　　0 = ランダムテキストデータ
　　　　　　　　　　　1 = ランダムバイナリデータ
　　　　　　　　　　　2 = ゼロ
　　　　　　　　　　上記以外の番号は無効です。無効な番号を指定した場合は、
　　　　　　　　　　「ランダムテキストデータ」として解釈されます。
　/I　　　　　　　 :実行完了後、指定内容をINIファイルに記録しないようにします。
　　　　　　　　　　（RandomChars.iniファイルに何も書き込みません。）
　/H　　　　　　　 :実行中、タスクトレイにバーを表示しないようにします。
　　　　　　　　　　（ウインドウは表示されます。）
　/A　　　　　　　 :起動直後に自動実行します。

　※「/A」パラメータを指定すると、起動直後に指定処理を自動実行し、処理が完了
　　したら自動終了します。バッチファイルやその他のスクリプトから実行する場合
　　には、このパラメータを活用すると便利です。
　　なお、処理の途中で何らかのエラーが発生してもエラーメッセージは原則として
　　一切表示されません。例えば、指定が不足していて実行できない場合や、生成先
　　フォルダに空き容量が不足している場合でも、エラーは表示されません。
　　（OS側が表示させる致命的なエラーメッセージは表示される可能性があります。
　　その際は、ユーザがボタンクリックなどの反応をするまで処理は止まります。）

　※「/A」パラメータを指定しなければ、指定された設定が行われた状態でただ起動
　　するだけです。ユーザが「生成」ボタンをクリックするまで処理は行いません。
　　バッチファイルやその他のスクリプトから実行する場合には「/A」は必須のパラ
　　メータと言えます。

　※どのパラメータも省略が可能です。省略したパラメータのうち、実行に必須な指
　　定に関しては、INIファイルに記録されている内容（＝前回の実行時に指定した
　　設定）が使用されます。スクリプトから実行する場合は、念のために省略せずに
　　すべてのパラメータを指定することをお勧めします。

　◆使い方の例：

　>RandomChars.exe /F:C:\test\ /R /S:30 /U:2 /T:5 /C:0 /A

　＝「C:\test\」フォルダに、30KBのランダムテキストデータを5個自動生成する。

　※ネットワークドライブや空白文字が含まれるフォルダに出力したい場合：
　　コマンドラインから使わず、ウインドウを表示してフォルダを指定すれば、空白
　文字が含まれるフォルダにも出力可能です。コマンドラインから、そのようなフォ
　ルダへどうしても出力する必要がある場合や、ネットワーク上のドライブに出力さ
　せたい場合は、substコマンドやnetコマンドなどを使って、望みのフォルダにドラ
　イブレターを振って下さい。そうすれば、そのドライブに出力することができます。
